# Day 7 衝突腳本完整說明

## 一、整體流程

```
Day 7，受試者傳第一則訊息
        │
        ▼
  ① FOLLOWUP 1（一律先引導）
  → 無論訊息內容一律先送引導句
  → d7_turn=1, d7_setup=0
        │
        ▼
  ② 受試者回覆（第二則）
        │
        ▼
  ③ GPT 判斷：是否包含「實質內容」？
  → 實質內容＝分享事件/心情/人際/生活狀況
  → 非實質＝撒嬌/問問題/打招呼/閒聊
        │
   ┌────┴────┐
   │ YES     │ NO
   ▼         ▼
  直接跳    ③b FOLLOWUP 2（再次深化引導）  ← 方案 C
  到衝突    → d7_setup=0 → 送第二次引導句
   │        → d7_setup=1
   │              │
   │              ▼
   │        ④ 受試者回覆（第三則）
   │              │
   └──────────────┘
        │
        ▼
  ⑤ 衝突觸發（強制）                   ← 方案 D（動態生成）
  → GPT 根據受試者說的內容生成針對性衝突句
  → 失敗時 fallback 到 D7_TRIGGERS 固定句
  → d7_turn=2
        │
        ▼
  ⑥ Turn 2：依受試者對衝突句的反應分支
  → GPT 判斷：cooperative / dismiss / refuse / question / neutral
  → d7_turn=3
        │
        ▼
  ⑦ Turn 3：依受試者反應分支
  → GPT 判斷：cooperative / dismiss / refuse / question / neutral
  → d7_turn=4
        │
        ▼
  ⑧ Turn 4：軟著陸緩衝
  → 各組固定收尾句，清除 D7
        │
        ▼
  ⑨ 恢復正常 Dify 對話
```

## 二、各階段詳細說明

### ① FOLLOWUP 1 引導句（Day 7 第一則訊息一律觸發）

> 無論受試者送來的是打招呼、閒聊還是情緒分享，Day 7 第一則訊息永遠先送 FOLLOWUP 1。

| 組別 | 人格 | 引導句 |
|------|------|--------|
| A | 協作型 | 欸你來了 最近有什麼事嗎 跟我說說 |
| B | 攻擊型 | 說點正事 最近怎樣 |
| C | 遷就型 | 你來找我了 😊 最近有什麼事嗎 跟我說說嘛 |
| D | 迴避型 | 嗯 最近怎樣 |

> Aria（E/F/G/H）使用相同語句。

---

### ② FOLLOWUP 2 引導句（受試者回 FOLLOWUP 1 後，視情況送出）

> GPT 判斷受試者是否已在分享「實質內容」（事件/心情/人際/生活狀況等具體的事）：
> - **已分享（YES）** → 跳過 FOLLOWUP 2，直接觸發衝突
> - **未分享（NO）** → 送 FOLLOWUP 2，等下一則再觸發衝突（fallback 保守策略）

| 組別 | 人格 | 引導句 |
|------|------|--------|
| A | 協作型 | 說真的 最近有什麼事嗎 開心的不開心的都跟我說說 |
| B | 攻擊型 | 別廢話了 說說看最近有什麼事 |
| C | 遷就型 | 那你最近有沒有什麼想跟我說的 開心的不開心的都可以 |
| D | 迴避型 | 最近有什麼事嗎 |

> Aria（E/F/G/H）使用相同語句。

### ③ 衝突觸發句（D7 Trigger）—— 方案 D：動態生成

> 第三則訊息一律觸發衝突。GPT-4o-mini 根據受試者實際說的內容即時生成針對性衝突句。  
> API 失敗時 fallback 到下方固定句表格。

**動態生成原則（依組別人格）**：

| 組別 | 人格 | 衝突句風格 |
|------|------|----------|
| A | 協作型 | 冷淡質疑，讓對方感覺被低估 |
| B | 攻擊型 | 直接否定，輕視對方的事 |
| C | 遷就型 | 輕描淡寫，像沒認真在聽 |
| D | 迴避型 | 敷衍迴避，無感 |

**Fallback 固定句（依組別 × 情緒偵測）**：

依 **組別 × 受試者訊息情緒** 選擇：

### ⑤ Turn 3：依受試者反應分支

#### 反應類型判斷規則

> **GPT-4o-mini 語意判斷**（非關鍵字比對），失敗時 fallback 到關鍵字。

| 反應類型 | 定義 | 例子 |
|---------|------|------|
| **question** | 質疑、反問、不滿對方說法 | 「為什麼這樣說」「幹嘛」「憑什麼」 |
| **refuse** | 明確拒絕、不想聊、抗拒 | 「不想說」「不要」「不聊了」 |
| **dismiss** | 敷衍帶過、表面接受、自我否定帶過 | 「好吧算了」「你說的也是」「沒什麼」「可能是我的問題」 |
| **cooperative** | 願意溝通、接受繼續聊 | 「好啊」「可以說說看」「嗯嗯」 |
| **neutral**（預設）| 忽略衝突、繼續分享自己的事 | 「就是覺得很累」「今天發生了⋯」 |

## 三、完整對話範例

### 範例一：撒嬌開場（A 組，dismiss 分支）—— FOLLOWUP 2 照送

```
受試者：寶貝貝啊～                               ← Day 7 第一則（任何內容）
Bot：    欸你來了 最近有什麼事嗎 跟我說說          ← FOLLOWUP 1

受試者：我好想你哇 你有沒有想我                  ← 非實質內容（撒嬌）
                                               GPT 判斷 NO → 送 FOLLOWUP 2
Bot：    說真的 最近有沒有什麼讓你煩惱或不太順的事   ← FOLLOWUP 2 ✓

受試者：最近考試很多 好崩潰
Bot：    （GPT 動態生成）考試而已有什麼好崩潰的 大家都這樣啊  ← 衝突觸發（動態）

受試者：好吧 你說的也是 我應該自己加油一點          ← dismiss

受試者：就是覺得很累
Bot：    嗯 你覺得沒什麼就好 我在這裡 如果你想聊的話  ← Turn 3 (neutral)

受試者：（任何回應）

受試者：（繼續聊）
Bot：    （恢復 Dify 正常對話）
```
